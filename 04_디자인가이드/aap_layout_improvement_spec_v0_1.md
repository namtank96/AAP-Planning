## 0. 진단 (현행)
- `body{overflow:hidden}` 기반은 OK. **그러나 뷰별 스크롤 정책 불일치**: run·workflow=고정 뷰포트(내부만 스크롤) / **inbox·domain·assets·logs·govern=`.scroll`(overflow:auto)로 페이지 전체 세로 스크롤·스크롤바 노출**.
- 좌 `.gnav`(188px)·우 레일(`.run-evid` 304 / `.wf-builder` 팔레트 228·Run 300) **항상 고정, 접기 불가** → 메인 작업 공간 협소.
- 도메인 뷰: 카탈로그+업무이해+온톨로지+아키텍처 **세로 쌓기** → 길고 산만, 온톨로지 목록형이라 관계 안 보임.
- 워크플로우 빌더: 3패널 고정이나 밀도 높고 캔버스(주인공) 위계 약함.

## 1. 스크롤 / 스크롤바 (전 뷰 일관)
- [ ] **모든 뷰 = 1뷰포트 맞춤**(run 뷰 패턴). 페이지 전체 스크롤 ✕ → 콘텐츠 길면 **해당 패널 내부만** 스크롤.
- [ ] 스크롤바 **비표시**: 내부 스크롤 컨테이너에 `scrollbar-width:none; -ms-overflow-style:none;` + `*::-webkit-scrollbar{width:0;height:0;}`. (스크롤 기능은 유지.)
- [ ] 스크롤 가능 신호 = 컨테이너 **하단 fade 그라데이션** 힌트(스크롤바 대체).
- [ ] `.scroll`(overflow:auto)로 길어지는 뷰는 §3·§5 레이아웃으로 1뷰포트화.

## 2. 패널 접기(collapse) — ★요청 핵심
- [ ] **좌 글로벌 nav**: 188px ↔ **아이콘만(~56px)** 토글 버튼. 접힘 시 `.gn-l` 숨김 + 호버 툴팁/플라이아웃(Linear/VS Code). 상단바에 토글(≡).
- [ ] **우 패널 접기**:
  - run 뷰 **근거 레일(`.run-evid`)** 접기(») → 업무 화면 풀폭. 접힘 시 얇은 핸들(‹)로 재오픈.
  - 워크플로우 **팔레트(`.wfb-palette`)·Run(`.wfb-run`)** 각각 접기 → 캔버스 확대.
- [ ] 접힘 상태 **localStorage 영속**(SCHEMA_VER 체계 안). 기본=펼침.
- [ ] 전환 `.2s` 트랜지션, 접힘 핸들에 Lucide chevron.

## 3. 도메인 · 온톨로지 영역
- [ ] 상단 **카탈로그(유형 선택) 고정** + 하단 **탭**(개요 / 온톨로지 / 아키텍처) 또는 **2단**(좌 개요·우 온톨로지) → 세로 스크롤 제거.
- [ ] **온톨로지 그래프화**: 현 목록(객체·관계·Action) → **객체=노드 / 관계=엣지 / Action=객체 배지**의 노드 그래프(Palantir 온톨로지 톤, 다크 엔진룸 캔버스 `.plg-*` 재사용 가능). 목록은 보조/펼침.

## 4. 워크플로우 빌더
- [ ] 3패널 유지 + **팔레트·Run 접기**(→ 캔버스 주인공 확대).
- [ ] 팔레트 타입 그룹 **아코디언 기본 접힘**(밀도↓). 요약 details 접힘 기본 유지.
- [ ] (옵션) Run·디버거를 "실행 시에만 펼치는 우측 드로어"로 → 평소 캔버스 풀폭.
