# AI 자동차 손해사정 심사·처리 시스템 — 업무 처리 워크플로우

> **대상 시스템**: [dmp-portal.vercel.app](https://dmp-portal.vercel.app/) > "AI 자동차 손해사정 심사·처리 시스템" 카드
> **시스템 설명**: 사건 접수·견적·대인·과실·처리 5단계를 AI 워크플로우로 자동화하고, **49개 추론 규칙**으로 일관된 판정 근거를 제공. 사정사는 AI 권고와 근거를 검토하기만 하면 된다.
> **문서 범위**: 실무자(손해사정사)의 심사 단계 사용 시나리오 → 팀장의 KPI 관리까지 전체 업무 워크플로우
> **작성일**: 2026-06-05

## 1. 전체 워크플로우 한눈에 보기

```
[사건 배당]
    │
    ▼
① 지급 심사 (Eligibility) ──⛔ DENY──▶ 면책 처리 (음주/무보험/시효)
    │  PASS                └⏸ REVIEW─▶ 수기 검토 / SIU(특별조사) 이관
    ▼
② 견적 산정 (Estimate) ── AI 사진분석 → 부위별 수리비 자동산출
    ▼
③ 대인 피해 (Injury) ── 상해등급·치료기간·합의금 AI 산정 (피해자 없으면 skip)
    ▼
④ 과실 산정 (Fault) ── 증거 기반 과실비율 판정 + 협상 전략
    ▼
⑤ 처리 방법 (Method) ── AI가 4개 시나리오 제시 → 실무자 최종 선택
    ▼
[합의서/보고서 자동 생성 → 결재 → 지급 (서류 접수 후 10일 이내)]
    ▼
⑥ KPI 관리 ── 팀장: 배당·종결률·절감액·규칙 트리거·감사로그 모니터링
```

- **실무자 화면**: 6탭 워크플로우 (지급 심사 → 견적 산정 → 대인 피해 → 과실 산정 → 처리 방법 → KPI 관리 + 데이터 뷰어)
- **팀장 화면**: KPI 관리 대시보드 + 사건 관리 + 규칙 엔진 관리 + 사용자 관리 + 감사 로그
- **AI 백본**: 온톨로지 추론 규칙 49개(R01~R49), 참조 DB 118,902건(차량/부품/서비스센터/렌터카/약관), KNIA 401개 사고유형

## 2. 실무자(손해사정사) 시나리오 — 심사 단계부터

### STEP 0. 사건 배당 받기

| 항목 | 내용 |
|---|---|
| 배당 방식 | 시스템이 업무량·전문성 기반으로 최적 배정 (15초 주기 실시간 동기화) |

| 실무자 확인 | 워크리스트에서 신규 배당(🆕 배지) 확인 → 사건 클릭 → 심사 시작 |

### STEP 1. 지급 심사 (Eligibility) — 보험금 지급 대상 자동 검증

사건을 열면 AI가 **3개 검증을 병렬 수행**(약 2.5초)하고, 실무자는 결과를 검토한다.

| 검증 카드 | AI가 수행하는 4단계 | 데이터 소스 |
|---|---|---|
| **가입자 본인 확인** | 신분증 OCR → 정책 DB 조회 → 계약자-청구자 일치 → 차량등록원부 대조 | 계약 DB, 면허 DB |
| **운전자 적합성** | 한정특약 조회 → 가족범위 매칭 → 운전자 신원 확인 → 면허 정상 여부 | 약관(가족한정 등) |
| **보험사기 패턴** | 3년 청구이력 → 관계인 교차조회 → FDS 23종 규칙 → 정비소/병원 조합 분석 | FDS 사기방지 시스템 |

**판정 결과에 따른 실무자 행동:**

| Verdict | 의미 | 실무자 액션 |
|---|---|---|
| ✅ **PASS** | 지급 대상 확인 | 다음 단계(견적 산정)로 진행 |
| ⛔ **DENY** | 음주·무보험·시효 등 면책 사유 | 면책 처리 (필요시 정부보장사업 청구 경로 안내) |
| ⏸ **REVIEW** | 의심신호 2개 이상 | 수기 검토 보류 → 사기점수 75점 초과 시 **SIU 의뢰** (지급 보류 + 증거 보전) |

> **사기 탐지(R20) 트리거 예시**: 6개월 내 3회 반복접수, 동일 정비소 3회 이상, 접수 후 72시간 뒤 최초 병원 방문(지연병원), 기준 대비 2.5배 치료기간(과잉진료), 적정액 대비 300% 과다청구

### STEP 2. 견적 산정 (Estimate) — 수리비 자동 산출

| 작업 순서 | 실무자가 하는 일 | AI가 해주는 일 |
|---|---|---|
| 1. 차량 특정 | 원산지(국산/외산) → 제조사 → 모델 → 연식 → 주행거리 선택 | 차량DB 60,329건에서 자동 매칭 |
| 2. 파손부위 입력 | 사고 사진 업로드 또는 8개 카테고리 트리에서 직접 선택 | 📸 사진 분석 → 파손 부위·정도(경미/중간/심각/전손 추정)·신뢰도 자동 판정 |
| 3. 견적 검토 | 부품비·공임비·도장비 테이블 확인 | 부품DB 48,637건 기반 통상가 산출, 연쇄손상 추정(R29), ADAS 캘리브레이션 비용 자동 포함 |
| 4. 확정 | **💾 견적 확정** 버튼 | 미수선(현금정산) 제안: 견적의 70~80% 현금 지급 옵션 |

**검토 포인트 (온톨로지 규칙 발동 내역 확인):**
- `[R03]` 품질인증부품 사용 시 OEM 공시가 25% 환급 인센티브
- `[R10/R30]` 외산차 배율(×1.6), 슈퍼카 배율(×3.0) 자동 적용
- `[R11]` 신차(3개월 이내) 감가상각 보상
- `[R02]` 수리비 ÷ 차량가액 > 70~80% → 전손 자동 판정

### STEP 3. 대인 피해 (Injury) — 인적 손해 산정

피해자가 없으면 자동 skip. 있을 경우:

| 작업 | 내용 |
|---|---|
| 피해자 목록 검토 | 피해자명·나이·상해등급·치료기간·합의금 테이블 (예: 45세 / 14급 / 35일 / ₩3,000,000) |
| AI 상해등급 산정 | 약관 기준 자동 계산 (경추염좌 14급, 늑골골절 8급 등 — 14등급 체계: 1급 2억 ~ 14급 1,200만) |
| 과잉진료 탐지 | 통상 치료기간 대비 실제 치료기간 비교 알림 |
| AI 합의 추천 | 등급·지역별 통상 합의금 벤치마크 → 합의금 확정 시 **합의서 자동 생성** |
| 쌍방 대인 | 양측 대인 접수 시 과실 상계 자동 계산 |
| 특수 케이스 | 사망사고(R25): 위자료 ₩8,000,000 자동 적용 + 유족 분배 순위(배우자>자녀) + 자동 에스컬레이션 |

### STEP 4. 과실 산정 (Fault) — 증거 기반 과실비율 판정

**증거 입력 (우선순위: 경찰보고서 > 블랙박스 > CCTV > 목격자):**

| 증거 | 처리 |
|---|---|
| 📸 사고 현장 사진 | AI 파손 감지 + 현장 환경(날씨·신호·도로) 분석 |
| 📹 블랙박스 영상 | 충돌 시점·속도·신호 상태 시각화 |
| 📋 경찰 보고서 | 법적 증거력 최고 — 약도·진술 자동 파싱 |
| 🎥 주변 CCTV / 👁️ 목격자 진술 | 객관적 증거 보강 (선택) |

**AI 과실 판정 → 협상 → 확정:**

1. **🤖 AI 판정**: KNIA 401개 사고유형 자동 분류 + 수정요소 적용
   - 예: 교차로 기본 50:50 + 신호위반 가산 +35% → **85:15 판정**
   - 수정요소: 신호위반 +15~20%, 중앙선 침범 +20%, 음주 +10~20%, 급차선변경 +15% 등
2. **협상 전략 3단계**: ① 타사 최초 주장 분석 → ② 손보협회 기준 중재안 제시 → ③ 증거 상충점 대조
3. **확정**: 💾 과실 저장 → ✅ 과실 확정 (분쟁 시 "협의 중" 상태 유지하며 협상 계속)

### STEP 5. 처리 방법 (Method) — AI 4개 시나리오 중 최종 선택

AI가 자사 손실 최소화 관점에서 **4개 시나리오 카드**를 자동 생성한다:

| 시나리오 | 성격 | 예시 |
|---|---|---|
| **① 최적안** ★ RECOMMENDED | AI 추천 (만족도 ★4.8/5) | 미수선 현금정산 — 자사 비용 ₩0, 빠른 정산, 고객만족 高 |
| **② 공격안** | 타사 청구 감축 극대화 | 과실 재협상 + 구상권 행사 |
| **③ 안전안** | 분쟁 리스크 최소화 | 제휴센터 수리 + 전액 보상 |
| **④ 전략안** | 특수 경로 활용 | 정부보장사업 청구, 일괄 패키지 합의 등 |

각 카드에는 **방법·자사 부담액·타사 청구액·대인/대물 처리·처리기간(7~60일)·장단점·AI 분석**이 표시된다.

**실무자 최종 액션**: `✓ 이 방안으로 진행` 클릭 → 합의서/손해사정 보고서 **자동 생성** → 결재 상신

**결재 라인 (금액 기준):**

| 보험금 규모 | 결재 라인 |
|---|---|
| ~100만원 (소액간편) | 담당자 → **팀장 전결** |
| ~500만원 (일반) | → 센터장 |
| ~2,000만원 (고액) | → 본부장 + 심사팀 검증 |
| 2,000만원~ (초고액) | → 심사위원회 |
| 사망 건 | 별도 심사 프로세스 (필수) |

> **SLA**: 서류 접수 후 **10일 이내 지급** 의무 (R15)

### 자동심사 vs 수동심사 분기 기준 (참고)

| 자동 진행 조건 | 수동 심사/보류 조건 |
|---|---|
| 지급 심사 전 항목 PASS | DENY → 면책 검토 |
| 대물만 + 수리비 < 100만원 | 대인 포함 또는 수리비 ≥ 100만원 |

| 사기 징후 0점 | 사기점수 ≥ 60점 → 현장조사, ≥ 75점 → SIU 의뢰 |
| — | 사망/중상 건 (자동 에스컬레이션) |

## 3. 팀장 KPI 관리 워크플로우

### 3-1. 일상 모니터링 루틴 (KPI 대시보드)

**실시간 배당 현황 (15초 주기 동기화):**

| KPI 카드 | 내용 | 활용 |
|---|---|---|
| **배당 건수** | 당일 총 배당 (일일 목표 175건) | 시간대별 진행곡선 확인 (10시 35건 → 18시 175건) |
| **진행 중** | 분석완료+협의중+진행중 | 적체 여부 확인 |
| **종결** | 당일 종결 건수 | — |
| **종결률** | 종결/배당 % (50% 이상 정상) | 진행 바로 즉시 식별 |
| **누적 절감액** | 타사 최초 청구액 vs AI 적정액 차이 (예: ₩47.8M, 절감율 24.5%) | AI 효과 정량 보고 |

**팀원 워크로드 모니터링:**

```
이현수 사정사       박서연 사정사       김민준 사정사
담당 17건 / 완료 11건   담당 14건 / 완료 9건   담당 19건 / 완료 13건
강점: 견적·처리        강점: 과실·대인        강점: 처리·대인
```

→ 담당 건수 편차 확인(예: 52건 vs 31건) → **재배정 판단**. 강점 태그 기반으로 사건 유형별 최적 배당.

## 5. 핵심 메시지

1. **실무자는 "판단"에 집중** — 조회·계산·문서작성은 AI가 수행하고, 사정사는 AI 권고와 근거(발동 규칙)를 검토·확정한다. (판단/의사결정 업무 비중 10% → 50%, 전체 처리시간 40~50% 절감 설계)
2. **이상 신호는 자동 차단** — P0 규칙(음주·사기·사망·시효)이 서킷브레이커로 작동해 실무자 실수에 의한 부당지급을 방지한다.
3. **팀장은 "예외와 추세"를 관리** — 정상 건은 시스템이 흘려보내고, 팀장은 민원·사기 플래그·워크로드 편차·절감액 추세만 보면 된다.
4. **모든 판정에 근거가 남는다** — 49개 규칙 발동 내역과 감사 로그가 사건별로 기록되어 민원·소송 대응 시 일관된 판정 근거를 제시할 수 있다.
