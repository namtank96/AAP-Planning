# AAP 사업화 전략

KT DS AAP(Agentic AI Platform) 사업화를 위한 자료 모음입니다. 일반론적 사업계획서가 아니라, 실제 보고와 영업에서 바로 쓰는 **예상 질문·답변**과 **근거 있는 전략 포인트**를 정합적으로 모았습니다.

## 기준 자료 체계

이 폴더의 사업화 문서는 단독 자료가 아니라 아래 기준 자료를 사업화 관점으로 해석한 것입니다. 아키텍처와 capability, 런타임 설명을 고칠 때는 먼저 이 자료와의 정합성을 확인합니다. 버전이 오르면 이 표만 갱신하고, 다른 문서는 역할명으로 참조합니다.

| 역할 | 기준 자료 (현행) | 무엇의 기준인가 |
|---|---|---|
| 기술·구조 캐논 | `../참고자료/AAP 아키텍처_260608ver..png` | 8계층 아키텍처의 시각 기준선 |
| 데이터 기반 캐논 | `../01_선제안서/aap_proactive_offering_v0.7_260616.html` | 8계층을 io라인·Loop 태그까지 풀어낸 상세 캐논 본체(4-2)와 Operating Loop(4-1) |
| 실행 데모 캐논 | `../03_프로토타입/D_회의/aap_meeting_runtime_builder_v0_58.html` | 업무 요청이 들어왔을 때 capability를 선택·조합하는 Runtime Builder 작동 기준 |
| 사업화 SSOT | `역량진단_확보전략.md` | 위 캐논을 보유 수준·확보 방식·리스크·진입 전략으로 해석한 본문 |

아키텍처는 PNG(시각)와 proactive 4-2(상세)에 함께 살고, Runtime Builder는 그 8계층이 한 업무 안에서 어떻게 조합되는지를 보여 주는 실행 인스턴스입니다.

## 규약

- 수치와 사실은 출처를 함께 답니다. 출처가 없으면 추정으로 표시하고 확정하지 않습니다.
- 약점은 숨기지 않습니다. 지식·시맨틱(L4)은 현재 자체 역량이 부족한 확보 과제로 둡니다.
- 확정 사실과 추정, 벤더 발표 수준 주장을 구분합니다.
