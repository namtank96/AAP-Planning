AGENTIC AI

Master Agent는 유능한 비서실장입니다.

요청을 받아 맥락을 파악하고, 필요한 전문가(Agent)를 부르고, 사람에게 한번 확인(HITL)받은 뒤 일을 처리하는 과정.

① 전체 처리 흐름

AX 컨설팅팀 현대영 책임님이 “한국무역협회 차세대 AX 무역플랫폼 구축사업 제안서 작성 회의”를 여는 상황으로 따라가 봅니다.

사람 요청

2
Master Agent · 맥락 분석

무슨 회의인지 이해하기

요청한 회의의 종류·목적·내용이 무엇인지 맥락을 분석합니다. (예: 제안서 작성 → 고객/마감/산출물 파악)

분석 절차 · Master Agent가 하는 일

  1. 1발화 파싱 · 요청 문장을 뜯어 회의 유형어("제안서·킥오프"), 고객사명, 산출물, 일정 표현 같은 핵심 키워드와 개체명을 뽑아냅니다.
  2. 2의도 분류 · 뽑아낸 단서로 이 회의가 어떤 종류인지(제안·킥오프·주간보고·설계리뷰·의사결정 등) 목적 카테고리를 추정합니다.
  3. 3개체 해소 · "한국무역협회·현대영 책임·AX 무역플랫폼"처럼 언급된 대상을 사내 인사·조직·프로젝트 DB와 매칭해 실제 누구·무엇인지 확정합니다.
  4. 4맥락 보강 · 요청에 빠진 정보(고객·마감·산출물)를 과거 회의 이력과 KMS 문서를 조회해 채워 넣습니다.
  5. 5슬롯 채우기 · 누가·언제·어디서·무엇을·왜·어떻게(6하원칙)로 구조화하고, 비어 있는 항목은 추가로 확인할 대상으로 표시합니다.
  6. 6신뢰도 판단 · 분류·정리 결과의 확신도를 따져, 불확실하면 사람에게 다시 물어(HITL) 잘못된 가정으로 진행되는 것을 막습니다.

3
Master Agent · 계획 수립

필요한 데이터 · 필요한 Agent 확인

회의를 잘 치르려면 어떤 자료(데이터)가 필요하고 어떤 전문가(Agent)를 불러야 하는지, 각각 왜 필요한지 근거를 들어 정리합니다.

계획 절차 · Master Agent가 하는 일

  1. 1작업 분해 · 확정된 회의 유형을 표준 작업 목록으로 펼칩니다. (예: 제안 회의 → 자료수집·제안서작성·일정조율·회의실·참석자추천 …)
  2. 2필요 데이터 식별 · 각 작업에 어떤 근거·과거 산출물이 필요한지 정의하고, KMS에서 관련도순으로 자료 후보를 추려냅니다.
  3. 3Agent 매핑 · 각 작업을 담당 Agent에 배정하고, "왜 이 Agent가 필요한지" 근거를 함께 붙입니다.
  4. 4실행 순서·의존성 정렬 · Agent 간 선후관계(예: 자료수집 → 제안서작성)와 동시에 돌려도 되는 작업(병렬)을 구분해 실행 순서를 짭니다.
  5. 5리소스·제약 산정 · 참석 인원·장비·근무 위치 조건을 따져 회의실 후보와 가능한 회의 시각 후보를 계산하고, 필수/선택 참석자를 추천합니다.
  6. 6누락·리스크 점검 후 계획안 패키징 · 빠진 Agent·데이터·참석자가 없는지 검증하고, 외부 발송·권한 같은 리스크를 표시한 뒤, 사람이 추가·제외·시각 선택을 할 수 있는 형태로 요약합니다.

→ 계획안을 ④ HITL① 계획 승인으로 넘김

선정 기준

자료 · 목적에 맞는 근거를 KMS 관련도순으로

Agent · 회의 유형별 필수 작업을 담당자에 매핑

예시 · 7명인데 10인실?

인원 7명+여유석 → 8인 이상

장비 타지역 참석 → 화상 카메라 보유실

위치 여러 층 근무 → 동선 최소 중간 층

4
★ HITL ① · 계획 승인

“이대로 진행할까요?”

실행 전에 현대영 책임님에게 분석·계획을 보여주고 확인받습니다. 추가하거나 제외할 Agent·데이터가 있는지 물어보고, 후보 회의 시각 중 하나를 사람이 직접 선택합니다.

주고받는 과정 · 🧠 Agent ↔ 🧑 사람

  1. 1🧠 Agent · 계획안 제시 — 불러올 Agent 목록, 쓸 데이터, 회의 시각 후보, 회의실, 참석자를 한눈에 보도록 요약해 보여줍니다.
  2. 2🧠 Agent · 수정 옵션 제공 — 추가하거나 제외할 Agent·데이터를 고를 수 있게 선택지를 함께 띄웁니다.
  3. 3🧑 사람 · 검토 & 수정 — 계획을 살펴보고 빠진 것/불필요한 것을 직접 추가·제외합니다.
  4. 4🧑 사람 · 회의 시각 선택 — 후보 시각 중 하나를 직접 선택합니다. (자동 확정이 아니라 사람이 결정)
  5. 5🧠 Agent · 수정 반영 & 재검증 — 사람이 바꾼 내용을 계획에 반영하고, 그로 인한 의존성·누락·충돌을 다시 점검합니다.
  6. 6🧑→🧠 승인 → 실행 트리거 — 사람이 "이대로 진행"하면 계획을 확정해 ⑤ 실행으로 넘기고, 보류·반려하면 계획 단계로 되돌려 다시 다듬습니다.

→ 사람이 통제권을 쥐는 핵심 단계 (Human In The Loop)

5
사람 승인 → 실행

현대영 책임님 동의 후 실행 시작

승인을 받은 Master Agent가 계획대로 Agent들을 가동합니다. (녹취·회의록·통역 등 회의 중 Agent는 6단계에서 가동되고, 여기선 회의 전 준비 작업이 중심입니다.)

실행 절차 · Master Agent가 하는 일

  1. 1실행 계획 전개 · 승인된 계획을 실행 가능한 작업 그래프로 펼치고, 각 Agent에 목표·입력·제약을 주입합니다.
  2. 2Agent 가동(디스패치) · 의존성 순서대로 Agent를 호출하고, 동시에 돌려도 되는 작업은 병렬로 실행합니다. (일정조율·회의실 예약·참석자 확정·자료수집 등)
  3. 3데이터 공급 · 각 Agent가 일하는 데 필요한 자료를 KMS·RAG로 찾아 전달합니다.
  4. 4결과 핸드오프 · 앞 Agent의 산출물을 다음 Agent의 입력으로 연결합니다. (예: 자료수집 결과 → 제안서 초안 작성)
  5. 5진행 모니터링 & 예외 처리 · 각 Agent의 상태를 추적하고, 실패·지연·빈 결과가 생기면 재시도·대체하거나 사람에게 알립니다.
  6. 6중간 산출물 취합 · 회의 시각 확정·초대장 발송·회의실 예약·사전자료/제안서 초안 등 생성물을 모아 회의 준비 완료 상태로 만듭니다.

→ 회의 준비 완료 후 ⑥ HITL② 회의 시작으로

6
★ HITL ② · 회의 시작·종료

회의 소집 → 진행 → 종료

회의가 소집되어 진행되고 마무리됩니다. 이때 현대영 책임님이 “회의를 시작한다 / 끝났다”를 직접 알려 녹취·회의록 Agent의 가동 시점을 사람이 통제합니다.

주고받는 과정 · 🧠 Agent ↔ 🧑 사람

  1. 1🧑 사람 · 회의 시작 신호 — "회의를 시작한다"고 직접 알려, 실시간 Agent의 가동 시점을 통제합니다.
  2. 2🧠 Agent · 실시간 Agent 가동 — 신호를 받아 녹취(STT)·통역·회의록 Agent를 동시에 켜서 발언을 실시간으로 전사·번역·정리합니다.
  3. 3🧠 Agent · 실시간 처리·구조화 — 전사 텍스트를 안건·결정·미결·액션 항목으로 구조화하고, 회의 중 쟁점이 생기면 자료수집 Agent를 추가로 호출합니다.
  4. 4🧑 사람 · 진행 중 개입(선택) — 안건 추가·자료 요청·발표자료 표시 등 회의 맥락을 직접 입력해 흐름을 조정합니다.
  5. 5🧑 사람 · 회의 종료 신호 — "회의가 끝났다"고 직접 알려, 실시간 Agent를 정지시킵니다.
  6. 6🧠 Agent · 종료 처리 & 산출물 정리 — 녹취·회의록을 마감하고 회의록 초안·액션아이템을 정리해, ⑦ 최종 승인 준비를 마칩니다.

→ 회의 진행 맥락을 사람이 직접 입력 (Human In The Loop)

7
★ HITL ③ · 최종 실행 승인

“메일 발송·KMS 기록을 수행할까요?”

외부로 나가는 마지막 행동(메일 발송·KMS 기록) 직전에 현대영 책임님에게 다시 한번 확인받습니다. 사람이 승인해야 실제로 전송·기록됩니다.

주고받는 과정 · 🧠 Agent ↔ 🧑 사람

  1. 1🧠 Agent · 최종 산출물 모으기 — 회의록·제안서·액션아이템 등 외부로 나갈 결과물을 한데 정리합니다.
  2. 2🧠 Agent · 발송안 제시 — 누구에게(수신자), 어떤 채널(메일·메신저)로, 어디 KMS에 기록할지 발송·기록 계획을 함께 보여줍니다.
  3. 3🧠 Agent · 미리보기 & 리스크 표시 — 누구에게 무엇이 가는지 미리보기를 제공하고, 외부 수신자·민감정보 등 되돌리기 어려운 리스크를 플래그합니다.
  4. 4🧑 사람 · 내용·수신자 검토 & 수정 — 산출물·수신자·문구를 최종 확인하고 필요하면 고칩니다.
  5. 5🧑 사람 · 최종 승인 / 반려 — "메일 발송·KMS 기록을 수행"을 승인하거나 보류·반려합니다.
  6. 6🧠 Agent · 승인 결과 처리 — 승인 시에만 실제 전송·기록(⑧단계)을 트리거하고, 반려 시 수정 단계로 되돌립니다.

→ 되돌리기 어려운 행동 직전의 안전장치 (Human In The Loop)

8
결과 전달 · 기록

참석자에게 공유 · KMS에 기록

승인을 받은 뒤 회의 결과가 각 참석자에게 전달되고, 관련 KMS 페이지에 자동 기록됩니다.

마무리 절차 · Master Agent가 하는 일

  1. 1채널별 발송 · 각 참석자·이해관계자에게 결과물을 메일·메신저 등 대상에 맞는 채널로 전달합니다.
  2. 2KMS 자동 기록 · 산출물을 관련 KMS 공간에 분류·태깅하고 메타데이터와 함께 등록해 검색 가능한 조직 자산으로 남깁니다.
  3. 3액션아이템 등록·후속 예약 · 회의에서 정해진 할 일을 담당자·기한과 함께 등록하고, 후속 미팅 예약·마감 리마인더를 설정합니다.
  4. 4전달 결과 확인 · 발송 성공/실패와 열람 여부를 추적하고, 실패 시 재시도합니다.
  5. 5완료 보고 & 링크 회신 · 현대영 책임님에게 "공유 완료 + KMS 기록 위치(링크)"를 회신해 마무리를 알립니다.
  6. 6자산화·학습 반영 · 이번 회의 유형·요청 문구·산출물 패턴을 사내 암묵지/이력으로 축적해, 다음 회의의 ② 맥락 분석 정확도를 높입니다. (피드백 루프)

↩ 축적된 학습이 ② 맥락 분석의 단서로 되돌아가 전체 흐름이 선순환합니다.

💡 한 줄 요약 · 요청(1) → 이해(2) → 계획(3) → HITL① 계획 승인(4) → 승인·실행(5) → HITL② 회의 시작·종료(6) → HITL③ 최종 승인(7) → 결과 공유·기록(8). 이 표준 흐름 위에서 사람의 확인(HITL)이 3번 들어가며, ③번 라이브 동작 탭에서 이 흐름이 실제 Agent들과 함께 동작하는 모습을 봅니다.
