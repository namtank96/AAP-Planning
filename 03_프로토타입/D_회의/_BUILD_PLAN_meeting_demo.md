## 0. 확정 사항

- **포지션 = C안 (절충형)** — 청중: 고객 **기술 실무자·아키텍트**.
  - 아키텍처-forward 중앙 엔진룸은 이 청중에겐 적합(보닛 열기 OK).
  - "절충"의 의미 = ① 좌측은 항상 **실제 회의 업무 화면/산출물**을 보여 엔진룸을 현실 업무에 접지(추상 박스 그래프 금지) ② AAP 내부는 **요청 시 펼침**(Focus 모달 "자세히 보기" + Header 설명 토글)으로 강요 아닌 탐색 ③ 14단계 × 4패널을 **한 번에 쏟지 않고 Phase로 나눔**.
  - **가드레일(손해사정 교훈)**: 중앙 노드는 전부 좌측의 구체 업무 산출물과 1:1로 묶일 것. "아키텍처 용어만 가득한 화면"(설계서 §14 금지)으로 흐르면 개념 브라우저가 됨.
- **기준 문서** = 같은 폴더 4종: 적용 검토 / 전처리 분석 v0.2 / 시나리오 스크립트 v0.1 / **화면 설계서 v0.1(주 구현 스펙)**.
- **산출물 규약** = 단일 HTML, 외부 라이브러리 0, 한글 경로. 버전 접미사(`v0_1`→`v0_2`…), 구버전은 보존(손해사정 방식).
- **파일명(예정)** = `aap_meeting_runtime_demo_v0_1.html` (이 폴더).
- **버전 관리(260618~)**: 현행본만 루트, 이전 버전은 `_archive/`로 이동·보관(삭제 ✕). 빌드마다 직전본 자동 이동. 현행=`aap_meeting_runtime_explainer_v0_18.html`, v0_1~v0_17은 `_archive/`. (기획 PDF·screen design·이 빌드플랜은 루트 유지)
- **중앙 다이어그램 골격 = A안 "계층 밴드 파이프라인"** (260617 ASCII 합의). 6개 AAP 계층이 가로 밴드로 고정(Customer Env→Data→Business Semantic→Agentic Core→Action&Integration→Operational Learning), Trust=charcoal 외곽 프레임. 요청이 위→아래 통과.
  - **아코디언**: 비활성 밴드=얇은 한 줄 스트립, **활성 밴드만 펼쳐져 점등**(높이 수용 + 점등 메커니즘).
  - **발산·수렴 살리기**(A의 정적 포스터화 방지): 밴드 전환마다 `▷발산`/`◁수렴` 마커 명시. **Agentic 밴드 = Step 4→5→6 걸쳐 내부 변신하는 미니 무대**(Task분해→5 Agent 부채꼴 발산→Auto/HITL/Blocked 3분기 수렴).
  - **밴드 색**(설계서 §10): Data=blue·Semantic=purple·Agentic=teal·HITL/Policy=amber·위험=red·Action=green·Learning=gold·Trust=charcoal.
  - **접지 규칙(C안 가드레일)**: 점등 밴드 focus 노드 ↔ 좌측 카드 1:1(◆Meeting Event↔"AAP 해석결과", Agent발산↔참석자·일정·자료 후보). 끊기면 개념 브라우저로 전락.
  - **Step→밴드 점등 맵**: 0=CustEnv+Trust / 1=Data(RawReq) / 2=CustEnv→Data(▷Context) / 3=Semantic(◆MeetingEvent◁) / 4·5·6=Agentic(TaskGraph→발산→3분기) / 7=Action(▣ExecPkg◁) / 8=Trust+Action(HITL) / 9=Action(▷Writeback Cal·Mail·Docs·Room).

## 1. Phase → 세션 분할

| Phase | 범위 | 포함 작업 덩어리 | Done 기준 |
|---|---|---|---|
| **P1 · Core Run 수직 슬라이스** | Step 0~9 (준비→요청→맥락→의미화→Task분해→**Agent 발산**→조율→실행패키지→HITL→Writeback) | 셸 레이아웃 / STEPS 0~9 / 좌측 4화면(요청·해석·패키지·HITL) / 중앙 발산·수렴 엔진룸 / 우측 해석패널 / 하단 로그(추가만) | 요청→Writeback까지 좌·중·우·로그가 **한 흐름으로 단계 진행**됨 |
| **P2 · 인터랙션·심화** | P1 위에 | 로그 클릭→중앙/우측 연동 / Focus 모달 틀1+Core 확대장면(Meeting Event·조율·HITL·Writeback) / 툴팁 12개 | hover 툴팁 + 자세히보기 모달 + 로그 연동 동작 |
| **P3 · Extended/Learning** | Step 10~12 | 좌측 회의진행·회의록 2화면 / 중앙 Meeting Run·수렴·Learning 노드 / 우측·로그 확장 | 회의 진행→회의록→Skill 후보까지 동작 |
| **P4 · Playback·폴리시** | Step 13 + 마감 | Playback 순차재생(손해사정 `runAapxAnim` 패턴 재사용) / 모드 토글(Core·Extended·Learning·Governance) / 색·정렬·오버플로우 폴리시 | 전체 경로 자동 재생 + 모드 토글 + 시각 마감 |

> **시간의 80%는 중앙 다이어그램**(P1·P3에 집중). 좌/우/로그/모달/툴팁은 설계서에 문구·구조가 거의 다 있어 "채우기".
> 손해사정 전례상 **각 Phase는 1차본 → 피드백 → 재정렬 2~3회**를 가정한다. v0.1이 바로 통과한 적 없음.
