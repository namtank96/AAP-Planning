## 2. 탭② AAP 구조 지도 — **시안으로 전면 교체** (가장 큰 작업)

### 교체 원칙

### 이식 사양 (시안 기준)
1. **섹션 HTML 교체** — `<section data-tab="1">` 본문을 시안 구조로:
   - `#loop` (6단계 Execution Loop 띠, 활성 단계만 점등)
   - `#msg` (한 메시지 헤드라인 + 4대 역량 태그) ← **지도보다 먼저, 위에**
   - `.board` = `#custEnv`(좌, Customer Environment 정적 흐림) · `#stack`(중앙, L1~L8) · `#trust`(우, Trust·Security 정적 흐림)
   - `#result` (이 단계 결과 한 줄)
   - 하단 `.capkey`(4대 역량 풀이 한 줄)
   - **STEP 선택 버튼은 넣지 말 것** — 데모는 상단 공유 Phase Rail이 구동. 시안의 `#steps`/`render(i)` 자체 버튼은 제거하고 **`render(i)`를 `curPhase`로 호출**.
2. **데이터 이식** — 시안의 `LAYERS`·`LOOP`(6단계)·`CAPS`(4대 역량)·`CUST`·`TRUST`·`STEPS[6]`(step별 `headline`·`caps`·`loop`·`core`·`active{layer:note}`·`result`)를 그대로 가져옴. (PHASES와 별도 상수로 둬도 무방. 내용은 시안에서 검증됨.)
3. **점등 규칙** — STEP마다 `active`에 있는 계층만 또렷(`.layer.on`), `core`는 강조(`.layer.core` teal), 나머지 8계층은 흐림. L3는 항상 `★핵심`(PNG 표기) 유지.
4. **setPhase 연동** — `setPhase(i)`에서 탭2 렌더 함수 호출 추가(기존 `renderRuntimeMap(i)` 자리 교체). STEP 1~6 바꾸면 Loop·헤드라인·역량·점등 계층·결과가 모두 그 STEP으로 갱신.
5. **제거 정리(회귀 안전)** — 더 이상 안 쓰는 것 삭제: `renderRuntimeMap` 및 `#eloop`/`.map-2col`/`.mv-card` 관련 CSS·마크업, 데이터 `LOOPB`(7단계)·`L2LOOP`·`VERB`·`FLOW_OUT`·`MAP_KW`·`MAP_SKELETON`·`activeStages`(탭2 외에서 안 쓰면). `MAP_POS`/`LAYER_FULL`/`LAYER_ROLE`도 탭2 전용이면 제거. **삭제 시 이를 참조하는 JS도 동반 제거**(과거 반복된 `null.innerHTML` throw 주의).
6. **고지** — 시안의 하단 `.foot` 한 줄(가정값)만. §0의 고지 단일화 원칙 유지(헤더 badge-val + 하단 val-panel).

## 3. 탭③ AAP 작동 시뮬레이션 — **"설계도" → "진짜 시연"**

### 진단
v0.46 탭3(좌 콘솔 `#simConsole` + 우 다크 `#simAside`[작동 흐름·결과·미니서브그래프 접힘])는 좌우를 톤으로 나눴지만 **콘솔이 "미리 구운 정적 카드 6장"이라 시연감이 없음.** 우측 다크 패널은 장식적.

### 방향 — 콘솔이 *제품처럼 처리되는 것*을 보여주고, AAP는 얇은 주석으로 강등
1. **콘솔을 주화면으로, 우측 다크 패널은 폐기/강등.**
   - `#simAside`의 큰 다크 카드 묶음(작동 흐름 + 미니 서브그래프) **제거.** AAP 작동은 **콘솔 섹션 옆 얇은 주석 한 줄**(`TWIN_ANNOT`)로만. 단, 주석을 **탭2 캐논 어휘로 갱신**: 예) "이 검토 큐 = 탭② **③ 통제·권한(L3 멀티에이전트 비교 → L7 사람 검토 게이트)** 결과." (계층·역량 명시.)
   - (선택) AAP 작동을 완전히 없애기 아쉬우면, 콘솔 하단에 **얇은 1줄 레일**(이 STEP의 6단계 Loop 위치 + 핵심 계층 + 역량)만. 별도 패널 ✕.
2. **★시연 핵심 = 누적(accumulation).** STEP 1→6 진행 시 콘솔이 **사건이 실제 처리되며 쌓이는 것**처럼 보여야 함:
   - **사건 처리 현황(누적) 패널** 신설 — STEP 1..현재까지 완료 항목을 한 줄씩 적재(담보 확인 → 조사 결과 → 과실 검토 큐 → 손해액 → 지급 후보 → 손해사정서). 현재 STEP 항목은 "처리 중"/new 강조. (소스 = STEP별 짧은 라벨 = `PHASES[k].name` + 결과 키워드.)
   - **검토 큐 누적** — 검토로 분리된 항목이 STEP 진행에 따라 쌓임(과실 검토 → 대인 손해 검토 → 보류·SIU). 현재 STEP에서 추가된 큐 항목 강조.
   - **손해사정서 초안** — STEP 6에서 누적 결과로 채워져 나타남(자동 90만 + 검토 171만 + 근거).
   - 즉 콘솔은 항상 **선택 STEP 시점까지의 누적 상태**를 보여줌(Phase Rail로 아무 STEP이나 점프해도 그 시점까지 누적). 현재 STEP의 SIM[i] 상세 카드는 "활성 작업 영역"으로 유지.
3. **데이터** — `SIM[i].user`(현재 STEP 상세) 유지 + 누적 라벨/큐는 `SIM[0..i]` 또는 STEP별 결과 키워드로 도출. `simAct`/`twLog`(Decision Log)/`twWarn`/`twDoc` 로직·ID 보존.
4. **시각** — 좌(밝은 제품 콘솔)이 화면의 主. 누적 패널 + 활성 작업 카드 + 액션 버튼 + Decision Log. 우측 다크 큰 패널 없음.

### 수용 기준 (탭③)
- [ ] STEP 1→6 넘기면 "사건 처리 현황"과 "검토 큐"가 **눈에 띄게 쌓인다**(정적 6장 ✕).
- [ ] AAP 작동은 얇은 주석 한 줄로만, 탭2 계층·4대 역량 어휘로 연결.
- [ ] 손해사정사가 실제 쓰는 업무 화면처럼 읽힌다(설명 패널이 아니라 제품).
