    AAP 산출물

    90만은 왜 자동이고, 171만은 왜 검토인가.

    AAP의 핵심은 금액을 낮추는 것이 아니라, 자동 지급해도 되는 항목과 사람 판단이 필요한 항목을 분리하는 것입니다.

    총 지급 후보
    ₩2,610,000 = 자동 ₩900,000 + 검토 ₩1,710,000
    자동 처리 후보₩900,000자동 지급결의 후보
    • 담보 확인 완료 — 약관상 보장 범위 내
    • 대물 손해 근거 명확 — 견적·수리범위 확인 가능
    • 부품·공임 적정성 확인 가능
    • 과실·사기 충돌 영향 낮음
    • 지급결의 자동 처리 후보
    검토 보류₩1,710,000손해사정사 / SIU
    • 과실비율 충돌 — 블박 20:80 vs 판례 30:70
    • 대인 합의금 300만 적정성 검토
    • 사기 리스크 78점 (조사 후보)
    • 6개월 내 유사청구 2건
    • 손해사정사 / SIU 검토 라우팅

    도메인 주석
    • 90만 / 171만은 SME · 계리 검증 전 시뮬레이션 값입니다.
    • 분기 로직의 타당성을 검증받기 위한 예시이며, 확정 성과가 아닙니다.
    • 사기 리스크는 지급 거절 근거가 아니라 검토 라우팅 근거입니다.

    이 처리가 AAP 아키텍처의 어디에서 발생하는가 — 고객 자산부터 8계층까지.

    사건 1건이 Customer Environment → 접수 → 자료연동 → 의미화 → 분해 → 충돌 → HITL → 실행 → 학습으로 캐논 전체를 통과합니다. 각 단계의 도메인 산출물을 함께 표시합니다. (아키텍처는 이미지가 아니라 컴포넌트로 재현)
