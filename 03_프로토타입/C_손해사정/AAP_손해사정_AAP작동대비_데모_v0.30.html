① 처리 방식별 Workflow

수기 · 챗봇 · RAG · 단일 Agent · AAP의 작동 흐름 비교. 같은 사건을 5가지 방식이 각각 어떤 입력을 받아, 어떤 기능을 거쳐, 어떤 결과를 내는지 도식으로 봅니다. AAP만 판단을 나누고 비교해 자동 처리와 사람 검토로 분기합니다.

② AAP Runtime View

손해사정 데이터가 AAP 안에서 판단·분기·실행되는 구조. 좌측 입력이 중앙 Reasoning Engine을 거쳐 우측 Feedback / Action Loop로 이어지고, 결과가 다시 기준·스킬 개선으로 환류됩니다.

Architecture Activation — Runtime 노드가 AAP 아키텍처 어디에 해당하는지(보조 뷰)

③ 단계별 AAP 작동

업무 단계별 AAP Agent · Tool · Data 작동 형상. 상단 Process Rail에서 선택한 단계의 업무 질문 → 손해사정 AAP Agent → Tool → External Data → 결과 반환 흐름을 봅니다.
