검증 요청 / 한계

이 데모는 숫자 성과가 아니라 분기 구조의 타당성을 검증받기 위한 것입니다. 도메인 · AAP 구조 · 한계 세 가지로 정리했습니다.

⚖ 도메인 검증
  • 90만 / 171만 분기 기준이 손해사정 실무상 타당한가?
  • 과실 10%p 차이를 Conflict Ticket으로 보는 것이 적절한가?
  • 사기 리스크 78점은 SIU 후보로 충분한가?
  • 대인 합의금 300만을 검토 보류 항목으로 보는 기준이 타당한가?
  • Decision Log 항목이 민원 / 소송 대응에 충분한가?
🧩 AAP 구조 검증
  • Agent를 과실 / 손해액 / 사기 / 약관으로 나누는 것이 적절한가?
  • Supervisor가 사건 복잡도에 따라 Agent를 나누는 구조가 타당한가?
  • A2A로 충돌을 표면화하는 방식이 실제 검토 이벤트로 유용한가?
  • HITL 기준이 자동 / 검토 / 보류를 나누기에 충분한가?
  • Decision Log와 Governance가 설명책임·감사 대응에 충분한가?
⚠ 한계
  • 실제 LLM / Agent / Ontology 미구동 — 결정론 시뮬레이션
  • 수치 SME · 계리 미검증 (90만 / 171만 · 78점 등은 가정값)
  • 원본 단일 데모가 정교하므로 비교 공정성 추가 검토 필요
오늘 검증받고 싶은 것은 숫자 성과가 아니라, 이 분기 구조(도메인 기준 → 충돌 표면화 → 차등 HITL → 근거 기록)가 손해사정 실무에서 타당한가입니다.
