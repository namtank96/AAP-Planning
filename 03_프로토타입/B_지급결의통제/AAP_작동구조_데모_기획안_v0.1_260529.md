# AAP 작동구조 체감 데모 — 기획안 v0.1

> 작성일 2026-05-29 · KT DS AX컨설팅팀
> 목적: 고객이 **"AAP가 어떻게 작동하는가"** 를 쉽고 직관적으로 파악하게 한다.
> 단순 기능 위젯이 아니라, **솔루션 아키텍처가 흐르는 모습**을 보여준다.

---

## 0. 한 줄 정의

세탁 클레임이라는 누구나 이해하는 사례를 통해, **AAP의 작동 구조(데이터→온톨로지→추론→결정→액션→학습)가 한 화면에서 흐르는 것**을 보여주고, 같은 구조가 내부통제 등 타 도메인에 그대로 복제됨을 증명한다.

---

## 1. 배경 — 무엇이 부족했나

직전 산출물 `aap_poc_demo_laundry_v0.7_ontology.html`은 **추론 한 노드만 확대한 위젯**이었다. 강점(LLM 대비 결정론·추적·재현)은 분명하나, 두 가지가 안 보였다:

1. **전체 판단 흐름과 구조** — 클레임이 들어와 결정·액션·학습까지 어떻게 도는가
2. **고객 관점의 문제→해결 매핑** — 어떤 기존 문제에 어떤 AAP 기능이 적용되는가

### 벤치마크 3종이 한 방향을 가리킨다

| 출처 | 보여주는 구조 | 핵심 표현 |
|---|---|---|
| **Palantir AIP Architect** | 좌→우 연결 노드 그래프: 데이터 소싱→데이터 처리→**온톨로지 모델링**→앱→분석. 노드 클릭 시 상세 구현 펼침 | "조각들이 어떻게 맞물려 돌아가는가" |
| **Palantir RaceOS** | ingest/transform → Andretti **Ontology** 매핑 → 이상·경보·액션 표면화 → 엔지니어 결정(HITL) → 피드백 환류. "single pane of glass" | "augment human expertise, not replace" · "인간 결정이 사이클의 중심" |
| **AAP 선제안서 v0.1 (캐논)** | 상단 Execution Loop: **Data → Semantic → Reasoning → Integration → Learning ↻** + 7-layer 스택 | 단일 통합 다이어그램 |

> **결론: 셋 다 동일 구조다.** 이 흐름이 "엔터프라이즈 AI가 작동하는 방식"의 정설이며, 우리 캐논 아키텍처와도 일치한다. 데모는 이 구조를 **세탁으로 만져지게** 만들면 된다.

## 2. AAP 작동 구조 (캐논) — 도메인 무관

AAP는 6단계 **Execution Loop**로 돈다. (= 선제안서 v0.1 상단 바와 동일)

```
 ① 데이터        ② 시멘틱/온톨로지     ③ 추론(Agent)        ④ 결정(HITL)      ⑤ 통합/액션      ⑥ 학습 ↻
 Data       →   Semantic        →   Reasoning      →   Decision     →   Integration  →   Learning
 ──────────     ────────────────     ───────────────     ────────────     ────────────     ───────────────
 흩어진 원천      엔티티·관계로 구조화     규칙+사례+LLM        사람이 최종 결정    실행·기록         결정을 자산으로
 통합·정제       (지식 그래프)          근거 추적 가능        AI는 증강·제안      single pane      조직 전체 환류
```

| 단계 | 하는 일 | 7-layer 매핑 | 신뢰 장치 |
|---|---|---|---|
| ① 데이터 | 원천 통합·정제 | 데이터/연동 | 출처 보존 |
| ② 온톨로지 | 엔티티·관계 모델링 (지식그래프) | 지식/시멘틱 | 의미 일관성 |
| ③ 추론 | 결정론 규칙엔진 + 유사사례 + LLM 설명 | Agent Core | **재현성·추적성** |
| ④ 결정 | 임계 초과 시 사람 결정 (HITL) | 경험/접근 | **인간 권한 유지** |
| ⑤ 액션 | 시스템 반영·통지·감사로그 | 데이터/연동 | 감사 추적 |
| ⑥ 학습 | 결정→정책 환류, 조직 전파 | AI 품질/평가 | 거버넌스 승인 |

**관통 원칙(전 단계 공통):** 좌측 = 고객 환경 연동 / 우측 = Trust·Security·Governance / 중심 = **인간 의사결정**.

## 3. 세탁 적용 — 작동 구조의 인스턴스

같은 6노드에 세탁 클레임을 흘린다. **노드마다 [기존 문제] → [AAP 해결]** 을 붙여 고객 관점을 뚜렷하게.

 ──────────────   ─────────────────   ─────────────────  ────────────    ──────────     ──────────────

 고객 동의서        ⟶ 관계(그래프)        "왜?" 추적·재현      자동 이관)        감사로그          반영 (선순환)
 ──────────────   ─────────────────   ─────────────────  ────────────    ──────────     ──────────────
 [문제] 정보가      [문제] 점주 감(感)·    [문제] 판단 근거     [문제] 본사       [문제] 수기      [문제] 한 점포
  매장에 흩어져      규칙이 암묵지로         불명확, 매번 답이     병목·기준이        응대로 누락·     경험이 조직에
  본사가 못 봄       머릿속에만 있음         달라 감사 불가        들쭉날쭉          편차 발생        안 쌓임
 [해결] 한 곳        [해결] 정책·사례를     [해결] 정책·사례      [해결] 근거 자동   [해결] 통지·     [해결] 이번 결정이

## 4. 내부통제 복제 — 같은 구조, 다른 도메인

**엔진은 그대로, 어휘만 바뀐다.** 지급결의통제(프로토타입 B)에 동일 6노드를 매핑:

| 노드 | 세탁 | 내부통제 (지급결의) |
|---|---|---|
| ① 데이터 | 매장 접수·처리이력 | 전표·결의·증빙·규정 |
| ② 온톨로지 | 소재·증상·사례·책임 | 계정·통제항목·위반유형·승인권자 |
| ③ 추론 | 정상범위·온도 규칙 + 유사사례 | 통제 규칙(한도·분리·증빙) + 과거 적발사례 |
| ④ HITL | 본사 클레임팀 결정 | 통제 담당·승인자 검토 |
| ⑤ 액션 | SMS·POS 반영 | 반려·경보·결재 라우팅 |
| ⑥ 학습 | 소재 정책 보강 | 통제 룰·임계 보강 |

> **고객 메시지:** "세탁에서 본 그 구조 그대로입니다. AAP는 도메인 솔루션이 아니라, 도메인을 갈아끼우는 **운영 지능 플랫폼**입니다." (세탁=교보재 / 내부통제=확장 증명)

## 5. 화면 설계 — 줌 아웃 ↔ 줌 인

핵심 인터랙션은 **하나의 캔버스에서 전체 흐름을 보다가, 노드를 눌러 안으로 들어가는 것** (Palantir AIP Architect 방식).

```
┌─────────────────────────────────────────────────────────────────────┐
│  [상단] Execution Loop 바 — ①②③④⑤⑥ 가 좌→우로 빛나며 케이스가 흐름      │
├─────────────────────────────────────────────────────────────────────┤
│  [줌아웃 캔버스]                                                       │
│    ①──→②──→③──→④──→⑤──→⑥─┐                                          │
│    데이터  온톨로지 추론  결정  액션  학습 │   ← ⑥에서 ②로 환류 화살표(루프) │
│         └──────────────────────────┘                                  │
│    · 각 노드에 [문제→해결] 토글 칩                                      │
│    · "케이스 흘려보내기" 버튼 → 점이 노드를 타고 이동 (작동 애니메이션)    │
├─────────────────────────────────────────────────────────────────────┤
│  [줌인 패널] — 노드 클릭 시 하단/우측에 상세                            │
│    ② 클릭 → 지식그래프(소재-증상-사례-정책 관계, Cytoscape)             │
│    ③ 클릭 → v0.7 추론 위젯 (왜?드릴다운 + What-if + 재현성)             │
│    ④ 클릭 → 본사 결정 화면(근거 자동첨부 + 임계 이관)                    │
└─────────────────────────────────────────────────────────────────────┘
```

**설계 원칙**
- **Single pane of glass** — 전체 흐름·이상·액션이 한 화면 (RaceOS)
- **인간이 중심** — ④ HITL을 시각적으로 강조, "AI는 증강·제안, 결정은 사람"
- **문제→해결이 노드마다 명시** — 고객이 "내 문제가 어디서 풀리는지" 즉시 파악
- **줌인은 기존 자산 재활용** — ③에 v0.7, ②에 지식그래프(B안)
