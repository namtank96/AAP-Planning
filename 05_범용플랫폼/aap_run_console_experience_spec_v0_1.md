# AAP 조작형 콘솔 — 목적 · 완성 기준 (북극성) v0.1

> **왜 이 문서**: 깜빡임·속도 같은 이슈를 *반사적으로* 쫓다 보니 "검증은 OK인데 여전히 이상" 이 반복됐다. 원인 = **완성 기준(definition of done)이 없어서**다. 이 문서가 그 기준이다. run 뷰 조작형 콘솔(채용·계약·이후 모든 도메인)의 **목표 경험을 못박아**, 구현·검증이 이 기준을 통과해야 "완료".
> 적용: `05_범용플랫폼/app/` run 뷰. 디자인 가이드(`04_디자인가이드`)·Contract v2·N3(evaluate)와 정합.

---

## 1. 목적 (한 줄)

> **요청이 들어오면 → AAP가 워크플로우를 만들고 → 사람이 *조종(steer)* 하면 AAP가 *근거(basis)와 함께 다시 분석*하고 → *책임 지점(HITL)* 에서만 사람이 확정한다. 채용이든 계약이든 *같은 표준*으로.**

- 핵심 차별: "정해진 대로 보는 정적 화면" ✕ → **사람 의도에 반응하는 조작형 도구**.
- 화면 = 업무 도구(사용자 친화)이지, AAP를 전시하는 데모 ✕.

---

## 2. 완성 기준 (이걸 다 통과해야 "완료")

### A. 케이스 진입 · 자동 흐름
- [ ] 케이스를 열면 **AAP가 스스로 단계를 진행**(사람이 phase 클릭 ✕).
- [ ] 단, **휙 지나가지 않는다**: 각 단계가 **최소 ~1.2초 머무르며**, 그 단계가 **"무엇을·무엇으로(구성요소)·어떤 근거로"**(데이터·룰·수치 basis)를 보여준 뒤 다음으로.
- [ ] HITL 게이트에서만 멈춰 **모달**로 사람 확정 요청.
- [ ] 이미 진행된 케이스는 완료분을 *즉시 채워* 보여주고 현재 지점에서 멈춤(진입 시 전 단계를 다시 재생 ✕).

### B. 의도 조종(steering) · 라이브 재분석
- [ ] steering(재가중·필터·임계·분기)은 **인라인 컨트롤**(chat ✕).
- [ ] 바꾸면 **비침투 재분석 카드**(전체 덮기·블러 ✕, 작업영역은 dim만) + 진행바 + 단계가 **각 단계 basis(수식·룰·임계 부등식)와 함께** ~3~4.5초.
- [ ] 끝나면 결과가 **부드럽게** 변화 — 순위는 FLIP 활강, 숫자는 트윈. 판정(계약)은 verdict 전환.

### C. 통제점(HITL)
- [ ] verdict/게이트가 검토·반려·승인-필요일 때만 **모달**(책임 지점). 그 외엔 자율.

### D. ★ 모션 · 렌더 원칙 (깜빡임 0의 정의)
- [ ] **어떤 갱신도 콘솔 전체를 부수고 다시 만들지 않는다.** 뼈대(스테이지 strip · steering 바 · 그래프 컨테이너 · 근거 레일)는 **마운트 유지**, **바뀐 부분만** 갱신.
- [ ] 갱신 = 제자리 업데이트 또는 FLIP/트랜지션(≤.5s). **요소 파괴→재생성으로 인한 번쩍임 = 0.**
- [ ] 모션은 **상태 전이에만**(가이드 §2): pulse=작동중·fade=등장·활강=재정렬·트윈=수치. 장식 모션 ✕.
- [ ] 그래프는 steering에 **반응**하되(엣지·노드 트랜지션) 통째 재그리기로 깜빡임 ✕.

### E. 시각 (1차 디자인 패스 = 충족, 유지)
- [ ] **1뷰포트**(페이지 스크롤 ✕, 패널 내부 스크롤) · 타이포 통일 · 네비 또렷 · 카드 1스타일 · 신규 hex 0.

---

## 3. 금지 (안티패턴 — 발견 즉시 결함)
- 콘솔 전체 `innerHTML` 교체로 인한 **전면 번쩍임** (← 현재 깜빡임 주원인).
- 단계가 **근거 없이 휙** 지나감 (auto-run 너무 빠름).
- steering 후 전체 덮는 오버레이/블러.
- 같은 의미에 다른 모션·다른 폰트·새 색.
- AAP를 전시·설명하는 톤(데모 티). 화면은 *업무 도구*.

---

## 4. 검증 = 이 기준으로
- 헤드리스/실사용에서 **A~E 체크박스**가 다 참인지. 특히 **D(전면 재건축 0)** 를 코드로 확인(갱신 시 surfHead/surfBody 통째 교체 여부).
- "검증 OK"의 정의 = 위 박스 통과. (단순 렌더됨 ✕)

---

## 5. 적용 메모 (구현 세션용)
- 현재 `core.js renderOpConsole()`이 `surfHead.innerHTML`·`surfBody.innerHTML`를 **통째 교체** → D 위반(깜빡임 근원). **부분 갱신 구조로 전환** 필요(뼈대 1회 마운트 + `updateRows()/updateVerdict()/updateGraph()` 분리).
- auto-run 타이밍(`revealOps` 단계 간격·`playNext` 지연)을 A 기준으로 상향 + 단계별 basis 노출.
- steering 재분석(`RE_T`·`runReanalysis`)은 B 충족(2차에서 근접) — 부분 갱신과 정합만.
