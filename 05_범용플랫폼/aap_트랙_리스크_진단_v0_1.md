## 1. 🔴 검증 순환성 — "자기일관성 ≠ 정확성"

**가장 중요.** evaluate-in-the-loop 게이트는 **LLM이 만든 DSL**이 **LLM이 만든 seed**를 **LLM이 단 expectedOutcome**으로 보내는지 검사한다. 생성자와 채점자(oracle)가 같은 모델.
- **잡는 것(가치 실재)**: slot drift·구조적 misroute·분기 누락 = *내부 불일치*. 실제 2결함 표면화·자동수정함.
- **못 잡는 것**: 모델이 도메인을 **일관되게 틀리면** 통과. 예—DSR 규제선을 50%로 잘못 알면 룰 `>50`도 seed `dsr=45→AUTO`도 같이 틀려 게이트 PASS.
- **결론**: "6도메인 전수 게이트 통과"는 **자기일관성·재현성 증명**이지 **정확성 증명이 아님**. 대외 설명 시 경계 필수.
- **닫는 법**: (완료) 모든 게이트통과 주장에 이 캐비엇 부착. (후속) 외부 oracle — SME 검수·실제 케이스 정답셋·규정 원문 대조 — 를 채점자로 분리하면 순환 해소.

## 2. 🔴 도메인 사실성 미검증 (SME 부재)

여신 route 실값: `creditGrade≥7 컷`·`dsrRatio>40 규제`·`보수선 35`·`경계 33~35`·`LTV>60`. 보조금: 자격·예산·중복수급 임계. **그럴듯하지만 SME 확인 0 = 가정값.**
- 손해사정 데모에서 `14급→12급` 한 값 오류가 정상건을 과잉으로 오탐했던 것과 **동일 함정**. 금융·공공은 숫자 틀리면 신뢰 즉시 손실.
- **용도 구분**: 골격 재사용 입증용으로는 충분. **고객 시연용이면 SME 1회 검수 필수.**
- **닫는 법**: (트랙 밖·핸드오프) SME 검수 전까지 산출물에 `수치=가정값(SME 미검)` 라벨 유지. → `packs_baked_v2/README.md`·본 문서가 라벨 보유.

## 3. 🟡 출발점 비판이 절반만 닫힘

트랙은 "새 업무 요청=키워드 골격"에서 출발 →
- **(B) 런타임**: 진짜 결정론으로 해결 ✅
- **(A) 저작**: 오프라인 하니스로 *증명*했으나 **앱의 '새 업무 요청' 기능은 여전히 키워드 기반**. 하니스→앱 연결은 *개념적 다리*(드롭인 핸드오프)일 뿐.
- **오인 주의**: "원래 문제 해결됨" ✕ → 정확히 "**진짜로 만들 수 있음을 증명 + 런타임 구현, 앱 저작 UI 연결은 미완**".
- **닫는 주체**: app/ 세션(드롭인 통합 + 새 업무 요청 UI를 하니스 산출 소비로).

## 4. 🟡 드롭인 계약 드리프트

- 드롭인은 한때 *주장*("≈3줄 머지·무회귀")이었으나 → **app-load smoke로 실측 전환**(`audit_track.js` C검사: caseData→`{outcome,basis,inputs,reproducible}` shape 일치).
- 잔여 리스크: 내가 Pack v2 계약에 6팩을 굽는 동안 **다른 세션이 계약(core.js·packs)을 동시 변경**. → C1~C6 정합 갭 현재 상태 미상.
- **닫는 법**: (완료) `audit_track.js`가 매 실행 산출물·참조·shape를 객관 재검증 → 드리프트 즉시 표면화. (후속) 실제 app/ core.js에 1팩 로드까지 가면 완전 실증.

## 5. 🟢 claim 과표현 → 스코핑(완료)

검증: flow가 동일하지 않음(계약A 13단계/3게이트 ~ 나머지 7). 공유되는 건 **아키타입+3분기 route+엔진**이지 단계 토큰이 아님. → 문서 "토큰 단위 공유" → "아키타입·3분기·엔진 공유, 단계 수 가변"으로 정정.

## 6. 🟢 결정론 범위 협소 → 스코핑(완료)

엔진은 `knowledge.route`만 실행. `thresholds`·`lookupTables`는 prose(미실행)이고 `dsrRatio` 같은 값은 엔진이 **계산하지 않고 입력 슬롯으로 받음**. → 증명된 결정론 = "**선계산된 입력에 대한 라우팅**"(아키텍처상 정상 — Connector/rule이 값 충전. `aap_connector_계약_스펙` §2). DSR 산식까지의 결정론은 미증명. claim은 이 스코프로.

## 8. 자동 감사 운용

- **결정론 층** `audit_track.js`: A 팩 smoke · B 드롭인 smoke · C 드롭인 app-load shape · D slot drift · E 문서↔산출물 참조/개수 · F 과표현·정직성 문구. blocking>0 → exit 1(cron/CI 게이트). 산출 `audit_report.json`.
- **판정 층** `aap-track-auditor` 에이전트(`.claude/agents/`): 스크립트가 못 보는 도메인 사실성·claim 정직성·신규 리스크만. **운용=온디맨드 확정**(cron ✕ — 기획 저장소라 변동 sporadic, 반복 LLM 비용 회피). 호출=`Agent(subagent_type:'aap-track-auditor')` 또는 드리프트 의심 시. 스크립트는 비용 0이라 상시 `node audit_track.js` 가능.
- **원칙**: 진단도 *결정 사다리* — 기계로 되는 건 기계가, LLM은 판단만. 진단 자체가 #1 순환성을 반복하지 않도록 결정론 검사를 1차로 둔다.
