## 변경 배경 — 정합성 검토 결과

현행 v0.1 HTML과 캐논 이미지를 계층별 대조한 결과, **두 관점은 설계상 상보적**(Section 4 = Layer View, 별첨 C = Domain/업무흐름 View)이고 대부분 일치했으나 **3개 갭**이 있었음. 이미지(Version 0.3)가 더 최신·정교한데 본문이 따라오지 못한 상태였음.

| # | 갭 | v0.1 본문 | 캐논 이미지 |
|---|---|---|---|
| 1 | Execution Loop | **5단계** Data·Semantic·Reasoning·**Integration**·Learning (칸 번호 5,5 중복 버그) | **6단계** Data·Semantic·Reasoning·**Decision·Action**·Learning |
| 2 | L3 Agent Core | 실행엔진·메모리·모델GW·도구·가드레일·관리 | + **오케스트레이션·멀티에이전트(Supervisor·A2A·MCP)** + **HITL 런타임 검토 게이트** |
| 3 | L1 / L7 소소 | L1 검토승인 워크스페이스 / L7 Self-Improving | L1 **API Gateway/SDK·Channels** / L7 **개인정보 관리** |

특히 **5단계 루프가 문제의 뿌리**였음 — Decision을 빼면 별첨 C의 [5]HITL·[6]Action 구분(자동/검토 분기)이 루프에서 사라짐. 6단계가 별첨 C와도 정확히 맞음.

## 적용 내역 (v0.2)

### [1] Execution Loop 5 → 6단계
- `데이터 → 의미화 → 추론 → 실행 → 학습` (5) → `데이터 → 의미화 → 추론 → **판단** → 실행 → 학습` (6).
- 영문 칸: Data ▶ Semantic ▶ Reasoning ▶ **Decision ▶ Action** ▶ Learning. ('Integration' 1칸을 Decision+Action 2칸으로 분리.)
- 칸 번호(주 계층 힌트) 중복 버그 교정: Data=5·Semantic=4·Reasoning=3·**Decision=7·Action=1**·Learning=6.
- Section 4 lead 문구도 "5단계 → 6단계", 스테이지명 갱신.

### [2] L3 에이전트 실행환경(★핵심) Agent Core 보강
- `+ 오케스트레이션 · 멀티에이전트` (Supervisor · Task Routing · A2A · MCP · 멀티에이전트 협업)
- `+ HITL 런타임 검토 게이트` (Runtime Review / Approval Gate) — teal 강조(`.arch-comp.hitl-gate` CSS 신설, 이미지의 부각 표현 정합).
- 별첨 C [5] Agentic Reasoning(Planning&Orchestration, HITL&Action Policy)과도 일치.

### [3] 소소 정합
- L1 `+ API Gateway / SDK` (외부 채널 연계 · Portal·Teams·Slack·Voice·Email·API).
- L7 `+ 개인정보 관리` (Privacy).

### [4] 신설 — Layer ↔ Domain 대응표 (Section 4 끝)
- 별첨 C가 "두 관점 상보"라고만 선언하고 표가 없던 것을 **Execution Loop 6단계를 축으로** 명시. (v310엔 유사 표가 있었으나 현행 v0.1엔 없었음.)
- 4열: Execution Loop / 계층 관점(Layer) / 업무 흐름 관점(Domain·별첨 C) / 선제안서 4대 역량.

## 미반영 (의도)
- L1 기존 '검토·승인 워크스페이스', L7 'Self-Improving'은 **삭제하지 않고 유지**(콘텐츠 보존). 이미지엔 다른 위치/명칭이나, 본문 내 정당한 요소라 추가만 함.
- Section 4는 Layer View 1종. 이미지의 "화면 2 · Layer Capability Map"(L3 상세 확대본)까지 재현하진 않음 — 필요 시 후속.
