## 변경 요약
1. **4-1 AAP Operating Loop** — 업무 관점 6단계(Data·Semantic·Reasoning·Decision·Action·Learning) + 단계별 운영 산출물 카드로 재정리. 순환·화살표 기호 제거(환류는 Learning 산출물로만 표현).
2. **4-2 AAP Platform Architecture** — 캐논 이미지(`참고자료/AAP 아키텍처_260608ver..png`) 정합. **세로 8계층(적층) + 상단 가로 Execution Loop 띠(순환) + 우측 Trust 횡단 스파인(전 계층 공통, 풀하이트)** 구조. L3 에이전트 실행 환경이 가장 크고 상세(Interface 칩 + Agent Core[실행엔진 입력›계획›행동›응답 미니플로우·오케스트레이션·메모리·모델 게이트웨이·도구 실행·에이전트 관리] + HITL 런타임 검토 게이트). **미니플로우 복원**: Loop 띠·L3 실행엔진·L5 데이터 파이프라인(수집›정제›정규화›변환)을 방향성(›)으로 표현. Customer Environment는 플랫폼이 아니므로 4-2에서 제거(4-3 공통 기반으로 이동). 외부 연계 문장·접이식 삭제.
3. **4-3 Capability Map** — 8영역 재나열(4-2 중복)을 폐기하고 **공통 기반 역량(Customer Data·Service Experience·Enterprise Integration·Trust·Governance·Infrastructure) vs 자사 주도·차별 역량(Business Semantic Modeling·Agentic Reasoning·Operational Learning)** 2계층으로 재구성. 메시지='어디가 자사 차별 지점인가', 하단에서 5장 수행 단계로 연결.
4. **5장 수행 모델** — Assess·Define·Compose·Connect·Operate 각 단계에 연계 Capability 표기 보강. note 순환 기호 제거·문구 교체.
5. **기호·해설 의존 제거** — 4장 화살표/순환 기호 전면 제거, 2-3·3-3 카드 용어를 4장 4대 축(업무 의미화·시스템 반영·실행 통제·운영 학습)과 정합. 2·3·5장 본문 연결 화살표 제거(2-2/3-2 비교 다이어그램은 중점(·)으로 치환, 구조는 유지).
