# 선제안서 변경 로그 — v0.2 → v0.3 (260612)

- **대상:** `00_현행_aap_proactive_offering_v0.3_260612.html` (현행)
- **직전본 보존:** `aap_proactive_offering_v0.2_260612.html`(무수정), `..._v0.1_260529.html`(무수정)
- **목적:** Section 4가 "통합 아키텍처(Layer)" 중심에 편중돼 *how before what* 으로 읽히던 문제를, **도식 중심 자연 흐름**으로 재구성. 서로 다른 축(Layer/Domain/Loop/역량)을 한 표에 억지로 잇지 않고 흐름으로 연결.

## 변경 배경
- v0.2에서 추가했던 **Layer↔Domain 대응표**가 Layer(구조)·Domain(업무흐름)·Execution Loop(시간축)·4대 역량(가치) **4개 다른 의도의 축을 한 표에 혼합** → "섞여 보인다"는 지적.
- 또 **Capability Map(역량)이 별첨에 묻혀** 있어, 업무(what)보다 구조(how)가 먼저 보였음.

## 적용 내역 (v0.3)
### [1] Section 4 재구성 — 흐름: 도메인 → 아키텍처 → 역량
- **4-1. Agentic Operation 도메인 (업무 흐름 8영역)** — 신설. [1]~[7] 흐름 + [8] 전영역 횡단 밴드. **Execution Loop(6단계)를 여기로 이동**(흐름=시간축이므로 구조에서 분리).
- **4-2. 계층 아키텍처 (8계층 보드)** — 기존 보드 유지. 旧 lead 문구·"한 장으로 읽는 법" note 제거(도식과 중복).
- **4-3. Capability Map (영역별 역량 · 요약)** — 旧 별첨 C를 본문으로 끌어올림. **상세 칩 폭발(10+/카드) → 아키텍처 보드 수준으로 압축**(영역별 핵심 역량 3~5 + ★자사 자산: Business Agents·AI:ON-U·MCP·BEAST·Antbot).

### [2] 旧 Layer↔Domain 대응표 제거
- 서로 다른 축을 억지로 잇는 표 대신, 4-1→4-2→4-3 **순서가 자연스럽게 연결**(좌 Customer Env=[1]·우 Trust=[8] 앵커가 N:M을 흡수). Loop는 4-1, 4대 역량은 Section 3에 귀속.

### [3] Execution Loop 계층-힌트 번호 제거
- v0.2의 칸 번호(주 계층 힌트)는 흐름축(4-1)에선 의미 없어 제거. 6단계(Data→Semantic→Reasoning→Decision→Action→Learning)는 유지.

### [4] 별첨 재번호
- 별첨 C(영역별 역량 상세) 본문 이동 → **sappC 제거**.
- 旧 별첨 D(수행 모델 상세표) → **별첨 C**, 旧 별첨 E(후속 논의) → **별첨 D**. 참조(`#sappD`,`#sappE`)·본문 링크 텍스트 갱신. 최종 별첨 = A·B·C·D.
