Past / Present / Future Capability Continuum

AAP는 갑자기 새로 시작하는 영역이 아니라, 과거 수행경험 위에 현재 솔루션을 조합하고 미래 Agentic 기술을 흡수하는 확장 구조입니다.

Past

이미 해온 수행 경험과 노하우

  • KMS / AICC 운영 경험
  • 그룹사·고객사 시스템 연동
  • 폐쇄망·보안·인프라 구축
  • ITO 운영 및 장애 대응
  • 콜센터·상담·지식관리 업무 이해

Present

지금 조합 가능한 솔루션과 기술

  • LLM / RAG / Vector Search
  • Agent Framework / Tool Calling
  • Workflow / API / RPA 연동
  • Copilot UI / Review UI
  • Monitoring / Audit / Observability

Future

Agentic 발전에 맞춰 흡수할 역량

  • 자기통제 / Guardrails 고도화
  • EvalOps / 품질 자동평가
  • Tool Registry 진화
  • 자율 실행 통제
  • Skill Marketplace / Agent 자산화
선제안 메시지 · KT DS는 Agentic AI를 처음부터 새로 시작하는 것이 아니라, 기존 SI·KMS·AICC·운영 경험을 기반으로 현재의 AI 솔루션을 조합하고, 미래 Agentic 기술을 단계적으로 내재화할 수 있습니다.
04

선제안용 핵심 차별화

프론티어 모델사나 범용 AI 플랫폼과 달리, 고객 업무·시스템·보안 환경에 들어가 실행까지 연결하는 SI형 Agentic Platform 제안입니다.

Why AAP

  • Data Platform 이후의 실행 계층
  • PoC/챗봇을 운영 업무로 확장
  • 업무 의미·정책·판단 기준을 Agent가 활용

Why SI

  • 고객 레거시 시스템과 깊은 연동
  • 업무 프로세스와 권한·승인 구조 이해
  • 구축·운영·개선 책임 인수

Why KT DS

  • KMS/AICC·SI·ITO 경험의 확장
  • Business Semantic과 Writeback 중심 제안
  • 폐쇄망·보안·운영 환경 대응 가능
